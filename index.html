How to use this page

  1. Pick a method — Caesar Shift, Vigenère, or Atbash — using the tabs in the workbench below.
  2. Set a key if the method needs one: a shift number for Caesar, or a word for Vigenère. Atbash needs no key at all.
  3. Type your message into the Plaintext box and press Encrypt to scramble it.
  4. Reverse it any time — paste ciphertext into the right-hand box and press Decrypt to get the original message back.

Everything runs locally in your browser. Nothing you type is sent anywhere.

Field Notes

Caesar Shift

The oldest trick in the book — literally used by Julius Caesar. Every letter slides forward by a fixed number of places. Shift 3 turns A into D. It's reversible because decrypting is just shifting backward by the same amount.

Vigenère

A Caesar cipher that changes its shift on every letter, driven by a repeating keyword. Harder to break by frequency analysis because the same plaintext letter can encrypt to different ciphertext letters.

Atbash

A mirror-image substitution with no key at all — A swaps with Z, B with Y, and so on. Simple, symmetric, and its own inverse: run it twice and you're back where you started.

Why this matters

Modern systems (AES, RSA, TLS) do the same job — reversible obfuscation of data — with vastly more mathematical rigor. Mastering the logic here is the foundation for reading real cryptographic code later.
